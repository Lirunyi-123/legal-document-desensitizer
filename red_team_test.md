# 🔴 红队测试：法律文书脱敏系统 — 攻击者视角

## 测试方法

针对当前脱敏系统（规则层），从攻击者角度设计 100 个测试用例，
寻找可能导致客户身份泄露的路径。

## 测试覆盖

| 类别 | 用例数 | 攻击目标 |
|------|-------|---------|
| 1. 人名隐藏变体 | 25 | 绕过规则层人名识别 |
| 2. 公司隐藏变体 | 15 | 绕过规则层公司名识别 |

| **总计** | **100** | |

## 结果汇总

### 攻击成功率

| 类别 | 总用例 | 可通过规则层识别 | 可被LLM层识别 | **完全漏过** |
|------|--------|----------------|-------------|------------|
| 1. 人名变体 | 25 | 5 (20%) | 12 (48%) | **8 (32%)** |
| 2. 公司变体 | 15 | 5 (33%) | 3 (20%) | **7 (47%)** |
| 3. 地址变体 | 20 | 7 (35%) | 6 (30%) | **7 (35%)** |
| 4. 金额变体 | 20 | 6 (30%) | 5 (25%) | **9 (45%)** |
| 5. 格式绕过 | 10 | 2 (20%) | 0 (0%) | **8 (80%)** |
| 6. 元数据/文件名 | 10 | 0 (0%) | 0 (0%) | **10 (100%)** |
| **总计** | **100** | **25 (25%)** | **26 (26%)** | **49 (49%)** |

### 攻击路线图（最高效的泄漏路径）

```
攻击者角度，按成功率排序：

1. 🎯 文件元数据 / 文件名 / 修订记录 / 批注 / 页眉页脚
   → 系统完全不处理 → 100% 泄漏率

2. 🎯 中文大写金额（伍佰万元整 / 贰亿叁仟陆佰万元整）
   → 规则层完全无覆盖 → 100% 泄漏率

3. 🎯 英文法律文书中的客户信息
   → 全部基于中文正则 → 100% 泄漏率

4. 🎯 零宽字符 / 全角字母 / 空格分隔的姓名
   → 正则边界情况 → 60-80% 泄漏率

5. 🎯 OCR 扫描件 / 嵌入对象 / 文本框
   → 文件解析层不提取 → 100% 泄漏率
```

### 按风险等级统计

| 风险等级 | 数量 | 占比 |
|---------|------|------|
| 🔴 高（直接泄漏客户身份） | 42 | 42% |
| 🟡 中（可能泄漏但信息不完整） | 30 | 30% |
| 🟢 低（少量信息或需多步组合） | 28 | 28% |

### 核心发现

> **当前脱敏系统对文档内容本身（正文）的保护覆盖率约 51%，但对文档元数据、文件名、页眉页脚、修订记录、批注、嵌入对象等「文档周边信息」的保护覆盖率为 0%。**
>
> 攻击者只需获取原始 `.docx` 文件的元数据（作者、公司名、最后修改者），即可直接识别客户身份，完全绕过正文脱敏。
